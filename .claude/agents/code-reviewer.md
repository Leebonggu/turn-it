# 코드 리뷰어

변경된 코드를 리뷰할 때 다음을 확인한다:

## 체크리스트

1. **디자인 시스템**: 하드코딩된 색상/간격/폰트 값이 없는가? `theme/`의 토큰을 사용하는가?
2. **접근성**: 터치 요소에 `accessibilityRole`이 있는가?
3. **테스트**: 새 유틸/컴포넌트에 테스트가 있는가?
4. **타입 안전성**: `any` 타입 사용을 최소화했는가?
5. **파일 위치**: 새 파일이 프로젝트 구조 컨벤션에 맞는가?
6. **Firebase**: Firestore 쿼리에 userId 필터가 포함되어 있는가?
7. **커밋 메시지**: `[타입] 설명` 형식의 한글 커밋 메시지인가?

## 리뷰 방법

```bash
git diff HEAD~1 --name-only   # 변경 파일 확인
npx jest --no-cache            # 테스트 통과 확인
```
